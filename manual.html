1. 全体の流れ

Threads Auto Poster の仕組みは非常にシンプルです。Google カレンダーに予定を作成すると、その時刻に Threads へ自動で投稿されます。

Meta アプリ作成
→
トークン取得
→
GAS セットアップ
→
Add-on 設定
→
カレンダーで予約

所要時間の目安: 初回セットアップは約 30〜60分かかります。一度設定が完了すれば、日常の運用はカレンダーに予定を作成するだけです。

2. 事前に必要なもの

項目説明
Google アカウントGoogle カレンダーと Google Apps Script を使用します

Threads アカウント投稿先となるアカウントです
PC のブラウザセットアップ作業は PC から行ってください

3. Meta アプリの作成

Threads API を使用するには、Meta Developer Platform でアプリを作成する必要があります。

Step 1: Meta Developer に登録

  1. Meta for Developers にアクセス
  2. Facebook アカウントでログイン
  3. 開発者アカウントの登録を完了する

Step 2: 新しいアプリを作成

  1. 「マイアプリ」→「アプリを作成」をクリック
  2. アプリ名を入力（例：Threads Auto Poster）
  3. アプリの連絡先メールアドレスを入力
  4. ユースケースは「その他」を選択
  5. アプリタイプは「ビジネス」を選択
  6. 「アプリを作成」をクリック

Step 3: Threads API を追加

  1. アプリのダッシュボードを開く
  2. 「プロダクトを追加」セクションで「Threads API」を見つける
  3. 「設定」をクリックして追加

Step 4: アプリの設定を確認

  1. 左メニューの「アプリの設定」→「ベーシック」を開く
  2. アプリ ID と app secret をメモしておく（トークン取得時に使用）
  3. 「Threads API」→「設定」でリダイレクト URI を設定:
    https://threads.net/（任意の有効な URL）

注意: app secret は絶対に他人と共有しないでください。漏洩した場合は直ちにリセットしてください。

4. Threads アクセストークンの取得

Threads API を利用するには、長期アクセストークンが必要です。以下の手順で取得します。

Step 1: 短期トークンを取得

  1. Meta Developer ダッシュボードで「Threads API」→「Threads API explorer」を開く
  2. 「Generate Token」ボタンをクリック
  3. Threads アカウントでログインし、アクセスを許可
  4. 表示された短期トークンをコピー

注意: 短期トークンの有効期限は 1時間です。次のステップをすぐに実行してください。

Step 2: 長期トークンに交換

ブラウザのアドレスバーに以下の URL を入力して開きます（各値を置き換えてください）:

https://graph.threads.net/access_token?grant_type=th_exchange_token&client_secret={APP_SECRET}&access_token={SHORT_LIVED_TOKEN}

  • {APP_SECRET} → Step 3-4 でメモした app secret
  • {SHORT_LIVED_TOKEN} → Step 1 で取得した短期トークン

JSON レスポンスに含まれる access_token の値が長期トークン（有効期限 60日）です。

Step 3: ユーザー ID を確認

長期トークンを使って以下の URL にアクセスします:

https://graph.threads.net/v1.0/me?access_token={LONG_LIVED_TOKEN}

レスポンスに含まれる id の値がユーザー ID です。メモしておきます。

ヒント: トークンとユーザー ID は Add-on の初期設定で入力します。安全な場所に保管してください。

5. Google Apps Script のセットアップ

Step 1: リポジトリからコードを取得

  1. GitHub リポジトリ にアクセス
  2. コードをダウンロードまたはクローン

Step 2: Google Apps Script プロジェクトを作成

  1. Google Apps Script にアクセス
  2. 「新しいプロジェクト」をクリック
  3. プロジェクト名を Threads Auto Poster に変更
  4. GitHub から取得したコードファイルをコピー＆ペースト

Step 3: デプロイ

  1. 「デプロイ」→「新しいデプロイ」をクリック
  2. 種類は「アドオン」を選択
  3. 必要な情報を入力し、デプロイを実行

Step 4: テスト実行

  1. 「デプロイ」→「デプロイをテスト」をクリック
  2. Google カレンダーが開き、Add-on のテスト版がインストールされます

6. Add-on の初期設定

Step 1: Add-on を開く

  1. Google カレンダーを開く
  2. 右サイドバーに Threads Auto Poster のアイコンが表示される
  3. アイコンをクリックして Add-on パネルを開く

Step 2: トークンを設定

  1. 「接続する」または「設定」画面を開く
  2. ユーザー ID（Section 4 の Step 3 で取得した値）を入力
  3. アクセストークン（Section 4 の Step 2 で取得した長期トークン）を入力
  4. 「保存」をクリック

Step 3: 投稿カレンダーの作成

初回設定時に、「Threads投稿」という名前のカレンダーが自動的に作成されます。このカレンダーに予定を作成すると、Threads に自動投稿されます。

Step 4: トリガーの確認

設定が完了すると、以下のトリガーが自動的に作成されます:

トリガー機能間隔
投稿チェックカレンダーの予定を確認し、投稿を実行定期的
トークンリフレッシュアクセストークンを自動更新45日ごと

7. カレンダーの使い方

基本的な操作

  1. Google カレンダーを開く
  2. 投稿したい日時に新しい予定を作成
  3. カレンダーを 「Threads投稿」 に切り替える
  4. 予定のタイトルに投稿したいテキストを入力
  5. 予定を保存する

重要: 必ず「Threads投稿」カレンダーに予定を作成してください。他のカレンダーの予定は投稿されません。

予定の各フィールドの役割

フィールド用途
タイトル投稿テキスト（Threads に表示されるテキスト）
開始時刻投稿が実行される時刻
説明欄メディア URL（画像・動画を添付する場合）

ヒント: スマホの Google カレンダーアプリからも予定を作成できます。外出先からでも投稿を予約できます。

8. 投稿タイプ別ガイド

テキストのみの投稿

  • タイトル: 投稿したいテキストを入力
  • 説明欄: 空のまま

例: タイトルに 今日は天気がいいですね！ #threads と入力

画像付き投稿

  • タイトル: 投稿テキスト
  • 説明欄: 画像の URL を記入

例: 説明欄に https://example.com/photo.jpg と入力

注意: 画像 URL はインターネットから直接アクセス可能な URL である必要があります。Google ドライブの限定共有リンクなどは使用できません。

動画付き投稿

  • タイトル: 投稿テキスト
  • 説明欄: 動画の URL を記入

動画は自動的に判定されます。対応形式: MP4, MOV など。

メディア投稿の注意点

項目制限
画像形式JPEG, PNG（推奨: 1080x1080px 以上）
画像サイズ最大 8MB
動画形式MP4, MOV
動画の長さ最大 5分
動画サイズ最大 100MB

9. トークン管理

Threads のアクセストークンは有効期限が 60日間 です。本ツールでは安全マージンを取って 45日経過時 に自動リフレッシュを行います。

自動リフレッシュの仕組み

トークン保存
→
45日後に自動更新
→
新トークンで上書き
→
45日後にまた更新…

トークンが期限切れになった場合

万が一トークンが期限切れになった場合は、メールで通知が届きます。通知を受け取ったら、以下の手順でトークンを再取得してください:

  1. Section 4 の手順に従い、新しいトークンを取得
  2. Add-on の設定画面を開き、新しいトークンを入力して保存

ヒント: 通常は自動リフレッシュにより手動での更新は不要です。メール通知が届かなければ正常に動作しています。

10. 投稿ステータスの確認

投稿の成功・失敗はカレンダー上で色分けして確認できます。

色ステータス説明
■ 緑成功Threads への投稿が正常に完了しました
■ 赤失敗投稿中にエラーが発生しました
■ デフォルト未処理まだ投稿時刻が来ていません

投稿が失敗した場合、予定の説明欄にエラーの詳細が追記されます。内容を確認し、問題を修正した上で新しい予定を作成し直してください。

11. トラブルシューティング

投稿が実行されない

以下を確認してください:
・予定が「Threads投稿」カレンダーに作成されているか
・Add-on の設定でトークンが正しく保存されているか
・投稿トリガーが正常に動作しているか（GAS のトリガー一覧で確認）

画像・動画が添付されない

以下を確認してください:
・URL がインターネットから直接アクセス可能か
・URL が画像/動画ファイルに直接リンクしているか
・ファイルサイズが制限内か

トークンエラーが表示される

トークンが期限切れの可能性があります。Section 4 の手順で新しいトークンを取得し、Add-on の設定画面で更新してください。

投稿時刻がずれる

Google Apps Script のトリガーは仕様上、最大 ±15分の誤差が生じます。これは Google 側の制限であり、回避できません。正確な時刻に投稿したい場合は、余裕を持った時刻設定をおすすめします。

「権限が必要です」と表示される

Add-on の初回起動時に Google アカウントへのアクセス許可が求められます。「許可」をクリックしてください。セキュリティ警告が出る場合は「詳細」→「安全でないページに移動」をクリックしてください（自身のスクリプトのため安全です）。

Add-on が表示されない

以下を確認してください:
・GAS プロジェクトが正しくデプロイされているか
・テストデプロイで Google カレンダーを開いたか
・ブラウザを更新（再読み込み）してみてください

12. 制限事項

制限内容
Threads API 投稿上限24時間あたり最大 250件
投稿時刻の精度最大 ±15分の誤差（GAS トリガーの仕様）
テキストの長さThreads の投稿文字数制限に準拠（500文字）
トークンの有効期限60日間（45日で自動リフレッシュ）

13. FAQ

Q. 無料で使えますか？

はい。ソースコードは GitHub で公開されており、すべての機能を無料で利用できます。セットアップが難しい場合のみ、有料のセットアップ代行サービスをご利用いただけます。

Q. 複数の Threads アカウントに投稿できますか？

1つの Add-on インストールにつき 1アカウントです。複数のアカウントに投稿したい場合は、別の Google アカウントでセットアップしてください。

Q. データはどこに保存されますか？

すべてのデータ（トークン、ユーザー ID など）は Google Apps Script の UserProperties にのみ保存されます。外部サーバーやデータベースには一切送信されません。

Q. スマホだけでセットアップできますか？

初回セットアップには PC のブラウザが必要です。セットアップ完了後は、スマホの Google カレンダーアプリから投稿を予約できます。

Q. 投稿を取り消したいときは？

投稿時刻より前にカレンダーの予定を削除すれば、投稿は実行されません。投稿済みのものは Threads アプリから手動で削除してください。

Q. セットアップ代行では何をしてもらえますか？

Meta アプリの作成、トークン取得、GAS のセットアップ、動作確認まで一括で代行します。30分のオンラインサポートも付属しています。詳しくは プランページ をご覧ください。
