1. 全体の流れ

Threads Auto Poster の仕組みは非常にシンプルです。Google カレンダーに予定を作成すると、その時刻に Threads へ自動で投稿されます。

Add-on をインストール
→
Threads アカウントを連携
→
カレンダーで予約投稿

所要時間の目安: 初回セットアップは約 5〜15分です。一度設定が完了すれば、日常の運用はカレンダーに予定を作成するだけです。

2. 事前に必要なもの

項目説明
Google アカウントGoogle カレンダーを使用します
Threads アカウント投稿先となるアカウントです
PC のブラウザ初回セットアップは PC から行ってください

3. Add-on の初期設定

Step 1: Add-on を開く

  1. Google カレンダーを開く
  2. 右サイドバーに Threads Auto Poster のアイコンが表示される
  3. アイコンをクリックして Add-on パネルを開く
  4. 初回は Google アカウントへのアクセス許可が求められるので「許可」をクリック

Step 2: Threads アカウントを連携

連携方法は 2 つあります。

方法 A: OAuth ログイン（おすすめ）
方法 B: トークン入力

方法 A: OAuth ログイン（おすすめ）

  1. 「🔗 Threads でログイン」ボタンをクリック
  2. Threads のログイン画面が表示されるので、アカウントでログイン
  3. アクセス許可を確認し「許可する」をクリック
  4. 「認証が完了しました！」と表示されたらウィンドウを閉じる
  5. Add-on パネルが自動的に更新され、接続済みになる

ヒント: OAuth ログインはボタン1回で完了します。トークンの取得や入力は一切不要です。

方法 B: トークンを手動入力

OAuth ログインが使えない場合は、アクセストークンを直接入力できます。

  1. Meta Developer Console でアクセストークンを取得（トークン取得の詳細手順）
  2. Add-on の「トークンで連携」セクションにアクセストークンを貼り付け
  3. 「保存して接続」をクリック

ポイント: ユーザー ID の入力は不要です。アクセストークンから自動的にユーザー ID が取得されます。

Step 3: 投稿カレンダーの作成

初回設定時に、「Threads投稿」という名前のカレンダーが自動的に作成されます。このカレンダーに予定を作成すると、Threads に自動投稿されます。

Step 4: トリガーの確認

設定が完了すると、以下のトリガーが自動的に作成されます:

トリガー機能間隔
投稿チェックカレンダーの予定を確認し、投稿を実行1時間ごと
トークンリフレッシュアクセストークンを自動更新毎週月曜 9時

4. カレンダーの使い方

基本的な操作

  1. Google カレンダーを開く
  2. 投稿したい日時に新しい予定を作成
  3. カレンダーを 「Threads投稿」 に切り替える
  4. 予定のタイトルに投稿したいテキストを入力
  5. 予定を保存する

重要: 必ず「Threads投稿」カレンダーに予定を作成してください。他のカレンダーの予定は投稿されません。

予定の各フィールドの役割

フィールド用途
タイトル投稿テキスト（Threads に表示されるテキスト）
開始時刻投稿が実行される時刻
説明欄メディア URL や返信設定（オプション）

ヒント: スマホの Google カレンダーアプリからも予定を作成できます。外出先からでも投稿を予約できます。

5. 投稿タイプ別ガイド

テキストのみの投稿

  • タイトル: 投稿したいテキストを入力
  • 説明欄: 空のまま

例: タイトルに 今日は天気がいいですね！ #threads と入力

画像付き投稿

  • タイトル: 投稿テキスト
  • 説明欄: #image: https://example.com/photo.jpg

注意: 画像 URL はインターネットから直接アクセス可能な https:// の URL である必要があります。Google ドライブの限定共有リンクなどは使用できません。

動画付き投稿

  • タイトル: 投稿テキスト
  • 説明欄: #video: https://example.com/video.mp4

返信コントロール

説明欄に以下を追記することで、返信を制限できます:

  • #reply: everyone — 全員（デフォルト）
  • #reply: accounts_you_follow — フォロワーのみ
  • #reply: mentioned_only — メンションされた人のみ

メディア投稿の制限

項目制限
画像形式JPEG, PNG（推奨: 1080x1080px 以上）
画像サイズ最大 8MB
動画形式MP4, MOV
動画の長さ最大 5分
動画サイズ最大 100MB

6. トークン管理

Threads のアクセストークンは有効期限が 60日間 です。本ツールでは安全マージンを取って 45日経過時 に自動リフレッシュを行います。

自動リフレッシュの仕組み

トークン保存
→
45日後に自動更新
→
新トークンで上書き
→
45日後にまた更新…

手動リフレッシュ

Add-on の設定画面（⚙）から「🔄 トークンをリフレッシュ」ボタンで手動更新もできます。

トークンが期限切れになった場合

万が一トークンが期限切れになった場合は、メールで通知が届きます。通知を受け取ったら:

  1. Add-on の設定画面を開き、「接続を解除」をクリック
  2. 「🔗 Threads でログイン」ボタンで再連携（または新しいトークンを入力）

ヒント: 通常は自動リフレッシュにより手動での更新は不要です。メール通知が届かなければ正常に動作しています。

7. 投稿ステータスの確認

投稿の成功・失敗はカレンダー上で色分けして確認できます。

色ステータス説明
■ 緑成功Threads への投稿が正常に完了しました
■ 赤失敗投稿中にエラーが発生しました
■ デフォルト未処理まだ投稿時刻が来ていません

また、投稿の成否に応じてイベントのタイトルに以下のタグが付与されます:

  • [POSTED] — 投稿成功
  • [FAILED] — 投稿失敗

投稿が失敗した場合、予定の説明欄にエラーの詳細が追記されます。内容を確認し、問題を修正した上で新しい予定を作成し直してください。

8. トラブルシューティング

「🔗 Threads でログイン」ボタンが表示されない

管理者が OAuth の設定（Client ID / Secret）を完了していない可能性があります。「トークンで連携」から手動で接続するか、管理者に連絡してください。

トークンが無効ですと表示される

以下を確認してください:
・トークンが正しくコピーされているか（前後にスペースが入っていないか）
・トークンの有効期限が切れていないか
・Threads テスターとして承認されているか（開発モードの場合）

投稿が実行されない

以下を確認してください:
・予定が「Threads投稿」カレンダーに作成されているか
・Add-on の設定でトークンが正しく保存されているか（設定画面で「✅ 接続済み」と表示されるか）
・投稿トリガーが正常に動作しているか（GAS のトリガー一覧で確認）

画像・動画が添付されない

以下を確認してください:
・説明欄の書式が正しいか（例: #image: https://...）
・URL が https:// で始まるインターネットからアクセス可能な URL か
・ファイルサイズが制限内か

投稿時刻がずれる

Google Apps Script のトリガーは仕様上、最大 ±15分の誤差が生じます。これは Google 側の制限であり、回避できません。正確な時刻に投稿したい場合は、余裕を持った時刻設定をおすすめします。

「アクセスをブロック」「Google の審査を完了していません」と表示される

テストモードのアドオンを使用している場合に表示されることがあります。「詳細」→「安全でないページに移動」をクリックしてください（自身のスクリプトのため安全です）。または、管理者に GCP の OAuth 同意画面で「テストユーザー」として追加してもらってください。

「time-based トリガーの数が多すぎます」と表示される

「保存して接続」を何度もクリックした場合に発生することがあります。Apps Script エディタでトリガー一覧を開き、不要なトリガーを手動で削除してください。または管理者が removeAllTriggers() 関数を実行することで一括削除できます。

9. 制限事項

制限内容
Threads API 投稿上限24時間あたり最大 250件
投稿時刻の精度最大 ±15分の誤差（GAS トリガーの仕様）
テキストの長さThreads の投稿文字数制限に準拠（500文字）
トークンの有効期限60日間（45日で自動リフレッシュ）

10. FAQ

Q. 無料で使えますか？

はい。ソースコードは GitHub で公開されており、すべての機能を無料で利用できます。セットアップが難しい場合のみ、有料のセットアップ代行サービスをご利用いただけます。

Q. 複数の Threads アカウントに投稿できますか？

1つの Add-on インストールにつき 1アカウントです。複数のアカウントに投稿したい場合は、別の Google アカウントでセットアップしてください。

Q. データはどこに保存されますか？

すべてのデータ（トークン、ユーザー ID など）は Google Apps Script の UserProperties にのみ保存されます。外部サーバーやデータベースには一切送信されません。

Q. スマホだけでセットアップできますか？

初回セットアップには PC のブラウザが必要です。セットアップ完了後は、スマホの Google カレンダーアプリから投稿を予約できます。

Q. 投稿を取り消したいときは？

投稿時刻より前にカレンダーの予定を削除すれば、投稿は実行されません。投稿済みのものは Threads アプリから手動で削除してください。

Q. セットアップ代行では何をしてもらえますか？

Add-on のインストール、Threads アカウントの連携、動作確認まで一括で代行します。オンラインサポートも付属しています。詳しくは プランページ をご覧ください。

11. 自分でホストする場合（開発者向け）

ソースコードをフォークして自分の環境で動かしたい開発者向けの手順です。一般利用者はこのセクションは不要です。

11-1. Meta アプリの作成

Step 1: Meta Developer に登録

  1. Meta for Developers にアクセス
  2. Facebook アカウントでログイン
  3. 開発者アカウントの登録を完了する

Step 2: 新しいアプリを作成

  1. 「マイアプリ」→「アプリを作成」をクリック
  2. アプリ名を入力（例：Threads Auto Poster）
  3. ユースケースは「その他」→ アプリタイプ「ビジネス」を選択

Step 3: Threads API を追加

  1. アプリのダッシュボード →「ユースケース」→「ユースケースを追加」
  2. 「Threads API にアクセス」を追加

11-2. GAS プロジェクトのセットアップ

Step 1: コードをデプロイ

  1. GitHub リポジトリ をクローン
  2. Google Apps Script で新しいプロジェクトを作成
  3. コードをコピー＆ペースト、または clasp push でデプロイ

Step 2: Script Properties を設定

  1. Apps Script エディタで左の ⚙ プロジェクトの設定 を開く
  2. 「スクリプト プロパティ」で以下の 2 つを追加:
プロパティ値
THREADS_CLIENT_IDMeta アプリの Threads app ID
THREADS_CLIENT_SECRETMeta アプリの Threads app secret

Step 3: Redirect URI を登録

  1. Apps Script エディタで initAdmin 関数を実行
  2. ログに表示される Redirect URI をコピー
  3. Meta Developer Console →「Threads API」→「設定」→「Redirect Callback URLs」に貼り付けて保存

Step 4: Threads テスターを追加（開発モードの場合）

  1. Meta Developer Console →「アプリの役割」→「Threads テスター」
  2. 利用者の Threads ユーザー名を追加
  3. 追加された側が Threads アプリの設定 → 招待で承認

Step 5: テスト実行

  1. Apps Script エディタ →「デプロイ」→「デプロイをテスト」
  2. Google カレンダーが開き、Add-on のテスト版がインストールされる

注意: Threads app secret は絶対に他人と共有しないでください。漏洩した場合は Meta Developer Console で直ちにリセットしてください。

11-3. トークンを手動で取得する場合

OAuth ログインを使わず、Meta Developer Console から直接トークンを取得する方法です。

Token Generator を使用

  1. Meta Developer Console → 「Threads API」→ 「Threads トークンジェネレーター」
  2. 「Generate Token」をクリック
  3. Threads アカウントでログインし、アクセスを許可
  4. 表示されたトークンをコピーし、Add-on の「トークンで連携」に貼り付け

ヒント: Token Generator で取得したトークンはそのまま使用できます。ユーザー ID は Add-on が自動で取得するため、手動で調べる必要はありません。
