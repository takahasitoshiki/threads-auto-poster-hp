プライバシーポリシー

1. はじめに

Threads Auto Poster（以下「本アプリ」）は、Google カレンダーの予定を Meta Threads へ自動投稿する Google Workspace Add-on です。本ポリシーでは、本アプリがどのようにユーザーの情報を収集・使用・保存するかを説明します。

2. 収集するデータ

データ項目目的
Threads ユーザーIDThreads API への投稿リクエストに使用
Threads アクセストークンThreads API の認証に使用
カレンダーイベント情報（タイトル・日時・説明）投稿内容の生成・スケジュール管理に使用
月次投稿カウントダッシュボードでの投稿統計表示に使用

3. データの使用目的

収集したデータは、以下の目的のみに使用します:

  • Google カレンダーの予定内容を Threads へ投稿すること
  • 投稿スケジュールの管理
  • アクセストークンの自動リフレッシュ（有効期限管理）
  • 投稿結果（成功・失敗）のカレンダーへの反映

4. データの保存場所

すべてのユーザーデータは Google Apps Script の UserProperties に保存されます。UserProperties はユーザーごとに隔離されており、他のユーザーからアクセスすることはできません。

本アプリは独自のサーバーやデータベースを持たず、外部にデータを保存することはありません。

5. 第三者へのデータ共有

本アプリは、ユーザーのデータを第三者と共有しません。データは以下の通信にのみ使用されます:

  • Threads API（graph.threads.net） — 投稿の作成・公開、レート制限の確認、トークンのリフレッシュ
  • Google Calendar API — カレンダーイベントの読み取り・更新

6. データの削除

ユーザーはいつでも以下の方法でデータを削除できます:

  • Add-on 設定画面の「接続を解除」ボタンをクリック → ユーザーID、アクセストークン、保存日時が即座に削除されます
  • Google Workspace から Add-on をアンインストール → すべての UserProperties が削除されます

詳細な手順はデータ削除ページをご覧ください。

7. セキュリティ

  • アクセストークンは Google Apps Script UserProperties に暗号化された状態で保存されます
  • すべての API 通信は HTTPS で暗号化されています
  • 本アプリのコードはオープンソース（MIT ライセンス）であり、誰でも確認できます

Privacy Policy

1. Introduction

Threads Auto Poster (the "App") is a Google Workspace Add-on that automatically posts Google Calendar events to Meta Threads. This policy explains how the App collects, uses, and stores user information.

2. Data We Collect

DataPurpose
Threads User IDUsed to make post requests to the Threads API
Threads Access TokenUsed for Threads API authentication
Calendar Event Information (title, date/time, description)Used to generate post content and manage scheduling
Monthly Post CountUsed for dashboard post statistics

3. How We Use Your Data

Collected data is used solely for the following purposes:

  • Posting Google Calendar event content to Threads
  • Managing post schedules
  • Automatic access token refresh (expiration management)
  • Reflecting post results (success/failure) back to the calendar

4. Data Storage

All user data is stored in Google Apps Script UserProperties. UserProperties are isolated per user and cannot be accessed by other users.

The App does not operate its own servers or databases and does not store data externally.

5. Data Sharing with Third Parties

The App does not share user data with any third parties. Data is used only in the following communications:

  • Threads API (graph.threads.net) — Creating and publishing posts, checking rate limits, refreshing tokens
  • Google Calendar API — Reading and updating calendar events

6. Data Deletion

Users can delete their data at any time using the following methods:

  • Click the "Disconnect" button in the Add-on settings — User ID, access token, and saved timestamp are immediately deleted
  • Uninstall the Add-on from Google Workspace — All UserProperties are deleted

For detailed instructions, see our Data Deletion page.

7. Security

  • Access tokens are stored in encrypted Google Apps Script UserProperties
  • All API communications are encrypted via HTTPS
  • The App's source code is open source (MIT License) and available for public review
